## Challenges / Blind Spots

1. **No live integration test.** Functional tests use `Spy`/`Null` gateways, so they validate the
   PHP contract but not the real orchestrator → bridge → ap-server runtime. Every 23.8 regression
   escaped the gates and was only caught in manual live testing.
2. **23.8 changed an artifact contract without a guardrail**, triggering a cascade of four fixes:
   - reachability broken (volume missing `/data/yamls` + `/data/worlds`) → 23.10
   - only the multidata persisted, patches/spoiler lost → 23.12
   - zip-in-zip when AP produced a bundle → orchestrateur #4
   - download saved as `.archipelago` (CORS `Content-Disposition` not exposed) → #32
3. **Unverified assumptions about Archipelago output** — single file vs whole dir; loose files
   vs a bundled `AP_*.zip`. Reality varies per game.
4. **CORS gotcha** — cross-origin `fetch` cannot read `Content-Disposition` unless exposed;
   same-origin functional tests never caught it.

**Root theme:** guardrails stop at build/unit/functional. Anything touching the multi-container
runtime (generation, launch, patch serving, downloads) was effectively untested until a human ran
the stack.

## Lessons Learned

- A green gate ≠ a working feature when the feature spans services. The most expensive bugs lived
  exactly in the inter-service contracts the gates don't exercise.
- Changing a runtime contract (the generation artifact) needs a contract test on **both** sides
  and a live verification before "done".
- Manual live testing (running `ArchipelagoServer` on the seed, inspecting the MinIO artifact,
  launching a real entry) was decisive — but it should be automated, not heroic.

## Action Items

1. **[Priority] Live weekly E2E smoke test** — a scripted integration test that runs
   generation → `session.generated` webhook → launch → and asserts: artifact is a flat zip of the
   real files, `/data/output` is restored on launch, member patches list the `.aplm`, and the
   seed loads in `ArchipelagoServer`. Owner: Dev/QA. Done = it fails on a contract regression.
2. **Cross-repo deploy+verify step** — any orchestrateur contract change requires a rebuild +
   live verification before the story is marked `done` (team agreement).
3. **Explicit artifact contract** — document the weekly output artifact (flat `.zip` of generated
   files) and assert the shape on both the orchestrateur (Go) and API sides.
4. **Download-endpoint checklist** — when adding a file-download endpoint, expose the needed
   response headers (`Content-Disposition`) via CORS and verify the cross-origin filename.
