## Ce qui a coûté / leçons

1. **`.gitignore` trop large = route silencieusement non commitée (incident majeur).** La règle `archipelago/` (repo microservice racine) matchait aussi `app/(public)/aide/archipelago/` → la page guide de 31.3 n'est jamais arrivée sur `develop`, et le **build local passait** (fichier présent sur disque) en masquant le trou. Détecté fortuitement deux stories plus tard.
   - *Leçon* : build local ≠ contenu versionné ; les patterns d'ignore non ancrés sont dangereux dès qu'un chemin applicatif réutilise le nom d'un repo séparé.
2. **Absence de CI = cause racine du point 1.** Aucun check automatisé sur les PR ; plusieurs PR mergées sans suite phpunit complète. Une CI buildant depuis un checkout propre aurait détecté la route manquante immédiatement.
3. **Trou de validation d'entrée récurrent.** Presque chaque story avait, en review, un défaut du même type : longueur non bornée → 500 (31.6, 31.8), lien 404 sur jeu `unavailable` (31.4) ou type d'étape inconnu (31.2), effet de bord pas post-commit (31.7). Corrigé en réaction à chaque fois.
   - *Leçon* : en faire une checklist proactive.
4. **Dette : shape d'étape dupliquée.** Ajouter `imageUrl`/`videoUrl` (31.5) a imposé de toucher la shape d'étape inlined dans de nombreux docblocks. Un type/value object partagé réduirait la friction.
